13  * <p>
14  * Like a BigDecimal, a SisuBigDecimal consists of an arbitrary precision integer unscaled value and a 32-bit integer
15  * scale. If zero or positive, the scale is the number of digits to the right of the decimal point. If negative, the

609  /**
610  * Returns the string representation of this SisuBigDecimal, using scientific notation if an exponent is needed. A
611  * standard canonical string form of the SisuBigDecimal is created as though by the following steps: first, the
612  * absolute value of the unscaled value of the BigDecimal is converted to a string in base ten using the characters
613  * '0' through '9' with no leading zeros (except if its value is zero, in which case a single '0' character is
614  * used). Next, an adjusted exponent is calculated; this is the negated scale, plus the number of characters in the
615  * converted unscaled value, less one. That is, -scale+(ulength-1), where ulength is the length of the absolute
616  * value of the unscaled value in decimal digits (its precision).
617  * <p>
618  * If the scale is greater than or equal to zero and the adjusted exponent is greater than or equal to -6, the
619  * number will be converted to a character form without using exponential notation. In this case, if the scale is
620  * zero then no decimal point is added and if the scale is positive a decimal point will be inserted with the scale
621  * specifying the number of characters to the right of the decimal point. '0' characters are added to the left of
622  * the converted unscaled value as necessary. If no character precedes the decimal point after this insertion then
623  * a conventional '0' character is prefixed.
624  * <p>
625  * Otherwise (that is, if the scale is negative, or the adjusted exponent is less than -6), the number will be
626  * converted to a character form using exponential notation. In this case, if the converted BigInteger has more than
627  * one digit a decimal point is inserted after the first digit. An exponent in character form is then suffixed to
628  * the converted unscaled value (perhaps with inserted decimal point); this comprises the letter 'E' followed
629  * immediately by the adjusted exponent converted to a character form. The latter is in base ten, using the
630  * characters '0' through '9' with no leading zeros, and is always prefixed by a sign character '-' ('\u002D') if
631  * the adjusted exponent is negative, '+' ('\u002B') otherwise).
632  * <p>
633  * Finally, the entire string is prefixed by a minus sign character '-' ('\u002D') if the unscaled value is less
634  * than zero. No sign character is prefixed if the unscaled value is zero or positive.
635  * <p>
636  * Examples:
637  * <p>
638  * For each representation [unscaled value, scale] on the left, the resulting string is shown on the right.
639  * <p>
640  * [123,0] "123"
641  * [-123,0] "-123"
642  * [123,-1] "1.23E+3"
643  * [123,-3] "1.23E+5"
644  * [123,1] "12.3"
645  * [123,5] "0.00123"
646  * [123,10] "1.23E-8"
647  * [-123,12] "-1.23E-10"
648  * <p>
649  * Notes:
650  * <p>
651  * There is a one-to-one mapping between the distinguishable BigDecimal values and the result of this conversion.
652  * That is, every distinguishable BigDecimal value (unscaled value and scale) has a unique string representation
653  * as a result of using toString. If that string representation is converted back to a BigDecimal using the
654  * BigDecimal(String) constructor, then the original value will be recovered.
655  * <p>
656  * The string produced for a given number is always the same; it is not affected by locale. This means that it can
657  * be used as a canonical string representation for exchanging decimal data, or as a key for a Hashtable, etc.
658  * Locale-sensitive number formatting and parsing is handled by the NumberFormat class and its subclasses.
659  * <p>
660  * The toEngineeringString() method may be used for presenting numbers with exponents in engineering notation, and
661  * the setScale method may be used for rounding a BigDecimal so it has a known number of digits after the decimal
662  * point.
663  * <p>
664  * The digit-to-character mapping provided by Character.forDigit is used.
665  * <p>
666  * Overrides:
667  * toString in class Object
668  *
669  * @return string representation of this BigDecimal.
670  */

678  * <p>

680  *

683  */

685  {

687  String str = number.toPlainString();
688  String[] parts = str.split("\\.");
689  if (parts.length == 1)
690  {
691  if (numDecimals == 0)

693  return parts[0];
694  }

696  {

698  }
699  }
700  else if (parts.length == 2)

704  return parts[0];
